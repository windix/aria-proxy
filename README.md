# Aria2 JSON-RPC Proxy

[中文](README_zh.md) | English

A Node.js proxy server that gracefully intercepts `aria2.addUri` JSON-RPC requests, archiving download URLs and their associated payload options (such as custom headers, cookies, and output paths) into a local SQLite database instead of securely handling the downloads directly!

This service provides a premium, fully-reactive **Vue.js 3** & **Tailwind CSS v4** Web UI to magically monitor your captured link requests and natively export them into a standard `aria2c -i` input text file at any time. Ideal for catching and forwarding heavy cross-site download requests generated by sophisticated browser extensions!

## Features

- **JSON-RPC Interception**: Effortlessly listens on `/jsonrpc` (acting natively as a mock Aria2 endpoint) and captures raw URLs alongside their complicated payload tracking rules.
- **RPC Authorization Protection**: Fully supports the standard Aria2 RPC Secret token pattern! You can strictly password-protect the proxy by passing an environment variable.
- **Smart Payload Normalization**: Traces injected `User-Agent`, `Referer`, and `Cookie` values exactly as passed within your JSON payloads even during heavy CORS filtering blockades.
- **Global User-Agent Overrides**: Optionally force all outgoing exports globally to use a strict, hardened custom `User-Agent`.
- **Premium Web Dashboard**: A very modern glassmorphism UI built dynamically via Vue 3 CDNs. Features interval-triggered auto-refreshing and nested payload inspection without complex build processes.
- **One-Click Export**: Flushes all collected pending requests symmetrically down into beautifully formatted `aria2c` multi-line download input files!

## Quick Setup

1. **Install bindings & external dependencies:**
   ```bash
   npm install
   ```

2. **Configure Environment Parameters:**
   Copy the provided `.env.example` to `.env` to easily manage custom proxy configurations!
   ```bash
   cp .env.example .env
   ```
   *Note: Using `.env`, you can flawlessly customize the proxy operation port, strictly secure the endpoint connection dropping by asserting an `ARIA2_RPC_SECRET`, or globally override all dynamically exported User-Agents using an isolated `USER_AGENT` definition.*

3. **Start the Proxy service:**
   ```bash
   npm start
   ```

4. **Access the Dashboard:**
   Open your browser natively and immediately navigate to the frontend UI:
   - [http://localhost:6800](http://localhost:6800)

5. **Connect Extensions:**
   Configure your browser extensions (or remote user-scripts) that normally point to Aria2. They will connect seamlessly to `http://localhost:6800/jsonrpc` where this proxy securely logs everything passively!

## Architecture Map

- `/src/server.js`: Main decoupled Express configurator and middleware bootstrapping router.
- `/src/jsonrpc.js`: Isolated, highly-secure JSON-RPC protocol parsing and credential verification handler.
- `/src/api.js`: Standard RESTful JSON endpoints powering the interactive frontend UI.
- `/src/db.js`: Underlying `better-sqlite3` synchronizer storing active requests persistently into your protected `/data/` directory block.
- `/public/`: Native, fast Vue 3 Single Page Application dashboard enforcing modern Tailwind interactions.

## Output Target Mapping

When you execute an export natively alongside the UI, the proxy smartly creates a payload modeled seamlessly into standard Aria2 Input formatting requirements:

```text
https://secured-remote-site.com/file.7z
  out=my-remote-downloads-folder/my-file_04.7z
  header=User-Agent: Mozilla/5.0...
  header=Cookie: session_id=1234
  header=Referer: https://tracker.com
```

You can then feed this text payload straight into any true background deployment script using `aria2c -i aria2_downloads.txt`!
